# Image Noise Forensics Tool

A command-line tool that analyses the noise patterns of an image to estimate whether it was captured by a real camera or generated by a diffusion / GAN model.

> **Not a definitive AI detector.** Results are heuristic. Do not use as sole evidence in any decision.

---

## How It Works

Every camera sensor imprints a unique noise signature — from photon shot noise, Bayer demosaicing, and the Photo Response Non-Uniformity (PRNU) pattern — that diffusion models cannot replicate faithfully. This tool extracts that noise residual and runs six signal-processing features against it:

| Feature | What it measures | Research basis |
|---|---|---|
| **EXIF completeness** | Presence of camera-specific tags (Make, Model, ISO, GPS, …) | Baseline |
| **Radial spectral energy** | High-frequency energy in the FFT spectrum; camera sensor noise raises the noise floor | FreqCross, 2025 |
| **Cross-channel noise correlation** | Bayer demosaicing creates R/G/B noise correlation absent in AI output | NASA / IEEE TIFS, 2025 |
| **Local variance structure** | Shot noise coefficient of variation + brightness-noise Pearson correlation (Poisson signature) | PRNU literature |
| **Texture sharpness** | Normalised Sobel gradient magnitude; AI images have characteristically soft edges | Edge-Enhanced ViT, 2025 |
| **Frequency anisotropy** | Angular energy variance in the mid-frequency ring; natural scenes are directional | Frank et al., ICML 2020 |

Each feature returns a score in [0, 1] where **0 = synthetic** and **1 = camera**. A weighted average produces the final score, and JPEG quality confidence attenuation is applied automatically when heavy compression is detected.

### Pipeline

```
Image → Loader → Preprocessor → Residual Extraction
      → Feature Extraction → Scoring → Visualisations + HTML Report
```

## Quick Start

```bash
# 1. Clone
git clone https://github.com/serious-engineer/Deepfake-Detector.git
cd Deepfake-Detector/forensics_tool

# 2. Install dependencies
pip install -r requirements.txt

# 3. Analyse an image
python main.py path/to/image.jpg

# 4. Verbose feature breakdown
python main.py photo.png --verbose

# 5. Custom output directory, skip report
python main.py img.webp --output ./results --no-report
```

### CLI Reference

```
usage: main.py [-h] [--output DIR] [--verbose] [--no-report] [--no-visuals] image

positional arguments:
  image              Path to the image file to analyse

options:
  --output, -o DIR   Root output directory (default: forensics_output/)
  --verbose, -v      Print individual feature scores
  --no-report        Skip HTML report generation
  --no-visuals       Skip forensic visualisation images
```

## Output

Each run creates a timestamped folder, e.g. `forensics_output/photo_20260418_152301/`:

| File | Description |
|---|---|
| `report.html` | Self-contained HTML report with all images embedded |
| `residual.png` | Wiener / Gaussian / Median noise residuals side-by-side |
| `spectrum.png` | FFT log-magnitude spectrum + radial energy profile |
| `heatmap.png` | Local noise variance heatmap (bright = noisy region) |
| `channel_noise.png` | Per-channel R/G/B Wiener residuals |

## Limitations

| Limitation | Notes |
|---|---|
| **Social media recompression** | Twitter/X, Instagram, WhatsApp strip EXIF and re-encode. Results degrade significantly. |
| **JPEG quality < 70** | Destroys high-frequency forensic signals; scoring confidence is automatically reduced. |
| **Post-processing** | Sharpening, denoising, upscaling, or colour grading can make AI images appear camera-like. |
| **Screenshots** | Desktop rendering adds noise that mimics camera sensor noise. |
| **Adversarial perturbations** | Imperceptible noise added intentionally can flip any heuristic detector. |
| **AI images printed and re-photographed** | Acquire real sensor noise; will appear authentic. |
| **No calibration dataset** | Score thresholds are heuristic (not trained on a validated set). |
| **Absence of synthetic signals ≠ proof of authenticity** | |
